Hackathon · Challenge B1

From learning to earning — one choice at a time

Roadrunner is an AI-guided, gamified learning journey. Like roadmap.sh, but every step offers branching choices — pivot from React to Swift on instinct, earn XP, and explore breadth-first toward real opportunity.

⚠ Human-in-the-loop 48hr MVP Greenfield Foundations locked

Concept

Something interactive like roadmap.sh with AI — every step the system gives you options to choose from, gamifying the learning journey. You can go completely off track from one field to another.
The ladder is basically disappearing for junior devs. The best is to know literally everything as much as possible and use AI to scale them.

This maps directly to B1 — The broken bridge from learning to earning: helping people move from learning into real opportunity through guided exploration, skill-building, and career navigation.

In scope (MVP)

  • Landing with B1 narrative
  • Supabase auth + onboarding
  • AI nodes with 2–3 choices
  • Pivot track action
  • XP, level, streak
  • Journey map visualization

Out of scope

  • Payments & leaderboards
  • Employer matching
  • Certificates & CMS
  • Offline mode
Landing → Sign in → Onboarding → Journey node → Choose path → Map / Pivot

Key Findings — Fix Before Build

SecuritySanitize AI markdown with rehype-sanitize; treat LLM output as untrusted
SecurityRate-limit /api/ai/next-node; keep LLM keys server-only
EngineeringAtomic flow: stream → validate → persist node → advance current_node_id → award XP
EngineeringPivot soft-archives orphaned subtrees via archived_at
EngineeringNested RLS on journey_nodes / journey_choices through journeys.user_id
DesignDefine empty, loading, error, success, auth-expired, and streaming-interrupted states
DesignNode hierarchy: lesson → primary choices → pivot as secondary link below
QASmoke: auth → onboarding → first node; RLS cross-user denial; AI fallback on parse failure
QALLM eval: structured JSON, 1–3 choices, fallback without corrupting tree

Implementation Order

  1. Scaffold Next.js + Tailwind + shadcn/ui in repo root
  2. Install Supabase, Zod, OpenAI/Gemini, rehype-sanitize
  3. Create Supabase project — tables, nested RLS policies, seed skill_catalog
  4. Middleware auth + /login + /onboarding
  5. Build /api/ai/next-node with validation, rate limit, fallback
  6. Journey node view + choice Server Action + XP updates
  7. Journey map (vertical SVG/list)
  8. Landing page with B1 narrative
  9. Deploy Vercel + manual QA matrix
